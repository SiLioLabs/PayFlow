# 🤝 Community Standards & Contributor Recognition

SoroScan is built by and for its community. We are dedicated to providing a welcoming, inclusive, and collaborative environment.

---

## 📜 Code of Conduct

All participants in the SoroScan ecosystem are expected to adhere to our Code of Conduct, which is summarized below:

- **Friendly and Patient**: We welcome contributors of all skill levels, from beginners to experts. Treat questions and feedback with patience.
- **Respectful**: Differences of opinion are natural. Keep debates technical and construct comments respectfully. Personal attacks, harassment, or dismissive behaviors are strictly prohibited.
- **Inclusive**: Encourage diverse viewpoints. Avoid gatekeeping or exclusionary language.
- **Collaborative**: Be open to collaboration. Help others get unstuck.

*To report violations, please contact the steering committee at `conduct@soroscan.io`.*

---

## 💬 Communication Norms

We use several channels to keep in touch and sync on development.

- **GitHub Discussions**: The primary channel for technical RFCs (Request for Comments), architecture proposals, and design debates.
- **Discord / Slack**: Used for real-time development sync, questions, and pair programming.
  - *Rule*: Keep support threads public. Do not DM maintainers directly for support unless they request it. This ensures knowledge is searchable by others.
- **GitHub Issues**: Reserved exclusively for bug reports and actionable feature requests.

---

## ⚖️ Resolving Conflicts

When disagreements arise regarding technical decisions:
1. **Focus on facts**: Use benchmarks, specs, or prototype code to support your arguments.
2. **Seek consensus**: Discuss options openly in GitHub Discussions/Issues.
3. **Escalation**: If consensus cannot be reached, the steering committee or project leads will make the final decision based on our architectural goals.

---

## 🌱 Welcoming New Contributors

- **Good First Issues**: We tag issues that require zero codebase-wide familiarity with `good first issue`. Maintainers will help guide you through these.
- **First PR Support**: We do not reject PRs for formatting or style mistakes. We guide you on how to resolve them or help push fixes directly to your branch.

---

## 🎓 Mentorship Opportunities

- **Office Hours**: Core maintainers host weekly office hours on Discord to help contributors understand the codebase, design contracts, or debug issues.
- **Pair Programming**: Request a pair programming slot on complex features to write code alongside maintainers.

---

## 🏆 Contributor Recognition

We value every contribution, big or small. SoroScan implements a multi-tier recognition framework:

```
[Contributor] ──> [Active Contributor] ──> [Core Reviewer] ──> [Maintainer]
```

### 1. The Hall of Fame
- We compile an active list of contributors in our `CONTRIBUTORS.md` file in the root.
- A bot automatically adds your GitHub profile to the list upon your first merged PR.

### 2. Contributor Swag & Rewards
- **Major Contributions**: Merging key roadmap items or uncovering critical security vulnerabilities makes you eligible for SoroScan swag packages (hoodies, stickers, shirts).
- **Stellar Community Fund (SCF)**: SoroScan regularly participates in the Stellar Community Fund. We share rewards with active community contributors who complete major deliverables.

### 3. Becoming a Maintainer
To request maintainer status (triage and merge write access):
- You must have at least **5 major merged PRs** or **15 code reviews** completed.
- You must demonstrate deep understanding of our architectural principles and community values.
- Submit a proposal in GitHub Discussions under the "Maintainership" category to get voted in by the steering committee.
